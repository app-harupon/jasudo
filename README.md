# ジャスドゥー — 迷わないTodo

スコアリング × マトリクス × カレンダー × セグメントタイマーを一直線でつなぐワンストップ型Todoアプリ。
「何をやるか」を毎回考えさせず、決まったことをぽんぽんと実行できる状態を仕組みで作る。

## 起動方法

静的サイトなのでサーバーは何でもOK。例:

```
python -m http.server 8788 --directory pon-todo
```

→ http://localhost:8788 を開く。データはブラウザの localStorage に保存される。

## 画面構成

| タブ | 役割 |
|---|---|
| マトリクス | 5×3アイゼンハワーマトリクス。重要度(高5.3/中4.2/低3.1)×緊急度(締切から自動算出1〜5)。スコアが高いセルほど色が濃い |
| カレンダー | 未スケジュールトレイから日付へドラッグ&ドロップ。「今日やる」リストはスコア降順 → ▶開始でそのままタイマーへ |
| タイマー | 合計時間を集中時間単位で自動分割。各セグメント: 俯瞰1分(16文字メモ) → 集中 → 休憩。全セグメント後「終わった/まだ」→ まだなら残り時間を再分割してループ |

## 主な仕様

- スコア = 重要度 + 緊急度(全15パターンが一意 → タイブレーク不要)
- 緊急度は締切日時から自動算出: 今日中5 / 3日以内4 / 1週間以内3 / 1カ月以内2 / それ以上1
- 半自動モード(トグル): 締切・所要時間・プロジェクトの段階(暫定: 序盤/中盤/終盤)から重要度を提案。最終決定は常にワンタップで上書き可
- 集中時間: 15/25/30/45/60分。休憩は集中時間の1/3が上限(超える選択肢はUIに出ない)
- 分割の微調整機能はあえて無し(迷わせない)
- 俯瞰メモは16文字制限・都度上書き(蓄積しない)
- タイマー実行中にページを閉じても、次回開いたとき一時停止状態で復元される

## ファイル構成

```
pon-todo/
├── index.html
├── css/style.css
└── js/
    ├── store.js     … データモデル・スコアリング・分割ロジック・永続化
    ├── ui.js        … モーダル・タスクフォーム・詳細・トースト
    ├── matrix.js    … 5×3マトリクス描画
    ├── calendar.js  … 月間カレンダー・D&D・今日リスト
    ├── timer.js     … セグメントタイマー(俯瞰→集中→休憩→完了確認→再分割)
    └── app.js       … タブ切替・初期化
```
